1 - What is the difference between synchronous and asynchronous file operations?
 Synchronous is a Blocking operation, the program waits until the file operation is finished, blocks the event loop. 
 Asynchronous is a non-Blocking operation, file operation runs in background, uses callback,promises or async/await.

 2 - When should you use file streams instead of reading the entire file? 
 Use file streams when the file is large or continous, and loading it fully memory is ineficient or impossible.


 4 - What are the common error codes in file system operations and what do they mean? 
 When file operations fail, Node.js returns an error object with a code.
   1.ENOENT — File or Directory Not Found  
   2.EACCES — Permission Denied
   3.EISDIR — Is a Directory
   4.ENOTDIR — Not a Directory
   5.EEXIST — File Already Exists
   6.EINVAL — Invalid Argument

 5 - How would you safely delete a directory with all its contents? 
 To safely delete a directory and everything inside it, the key ideas are: double-check the path, preview what will be deleted, and use the right command for your OS/tool. 

 6 - Explain the concept of piping in streams with an example. 
 Piping in streams is a clean, memory-efficient way to connect one stream’s output directly to another stream’s input—so data flows chunk-by-chunk without you manually handling buffers.

 7 - Why is it important to handle errors in file operations? 
 Handling errors in file operations is crucial because file systems are unpredictable—things can go wrong due to permissions, missing files, disk issues, or concurrent access. If you don’t handle errors, your app can crash, lose data, or behave incorrectly.

 8 - What is the difference between writeFile and appendFile methods? 
 writeFile() replaces file content, while appendFile() adds data without deleting existing content.
